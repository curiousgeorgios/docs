## Import from Asana[​](#import-from-asana "Direct link to Import from Asana")

> **Role**: Workspace admins

<Check>
  To import issue types from Asana, make sure the [Work item types](/core-concepts/issues/issue-types) feature is enabled in your Plane workspace.
</Check>

To import Asana issues to a Plane project, follow these steps:

1. Click the **∨** icon next to your workspace name on the sidebar and select **Workspace Settings**.

2. Select **Imports** on the right pane and click the **Import** button in the Asana section. ![Import from Asana](/images/importers/importers/asana/import-asana.webp)

3. In the **Asana to Plane Migration Assistant** screen, enter your **Personal Access Token** to allow Plane access to your Asana account. ![Connect Asana](/images/importers/importers/asana/asana-plane-migration-assistant.webp)

4. Click the **Connect Asana** button to link the accounts.

5. Click the **Import** button under the **Imports** section. ![Import Asana](/images/importers/importers/asana/import-asana-data.webp)

6. **Configure Plane** Select the Plane project where you want to import your Asana data and and click **Next**. ![Configure Plane](/images/importers/importers/asana/configure-plane.webp)

7. **Configure Asana** Choose the workspace and project in Asana from where you want to import data. ![Configure Asana](/images/importers/importers/asana/configure-asana.webp)

8. **Map states** Map **Asana sections** to their equivalent **Plane states**. ![Map states](/images/importers/importers/asana/map-states.webp)

9. **Map priorities** Map the **Asana priorities** to the corresponding **Plane priorities**. If there's no match, select **None** in the **Plane priorities** list. ![Map priorities](/images/importers/importers/asana/map-priorities.webp)

10. **Summary** Review the mappings and make any changes if needed. Click **Back** to adjust, or click **Confirm** to start the migration. ![Review mappings](/images/importers/importers/asana/import-summary.webp)

11. The data migration begins and takes a few minutes to complete depending on the number of issues in your Asana workspace. ![Migration complete](/images/importers/importers/asana/import-complete.webp)

12. Once it's done, go to **Work items** in your Plane project to confirm that the data import is successful.

## Imported entities[​](#imported-entities "Direct link to Imported entities")

Here’s a quick look at what gets imported during the migration from Asana to Plane:

| Asana                           | Plane                               | Notes                                                                                                                                           |
| ------------------------------- | ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Labels                          | Labels                              |                                                                                                                                                 |
| Status                          | States                              |                                                                                                                                                 |
| Issue priorities                | Priorities                          |                                                                                                                                                 |
| Users                           | Users                               |                                                                                                                                                 |
| Issues                          | Work items                          |                                                                                                                                                 |
| Relations                       | Parent                              | Includes only parent-child relationships                                                                                                        |
| Issue comments                  | Work item comments                  | Includes username and timestamp. If you skip user import during migration, comments will show the name of the user who performed the migration. |
| Issue attachments               | Work item attachments               |                                                                                                                                                 |
| Reporter                        | Created by                          |                                                                                                                                                 |
| Created                         | Created at                          |                                                                                                                                                 |
| Assignee                        | Assignees                           | If you skip user import during migration, this will be blank.                                                                                   |
| Issue types                     | Labels \| Prefix in Issue title     |                                                                                                                                                 |
| Images in the Issue description | Images in the Work item description |                                                                                                                                                 |
| Summary                         | Work item title                     |                                                                                                                                                 |
| Start date                      | Start date                          |                                                                                                                                                 |
| Due date                        | Due date                            |                                                                                                                                                 |
| Linked Issues                   | Links                               | Includes backlinks to the original Asana issue.                                                                                                 |

## Sync Asana to Plane[​](#sync-asana-to-plane "Direct link to Sync Asana to Plane")

After the import, if there are any new or updated issues in Asana, you can easily sync these changes to Plane:

1. Go to **Workspace settings**.
2. Select **Imports** on the right pane.
3. Click the **Re run** button next to the project you want to sync. ![Sync Asana](/images/importers/importers/asana/rerun-import.webp)
